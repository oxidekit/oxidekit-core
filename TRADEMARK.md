# OxideKit Trademark Policy

> Last Updated: January 2026

This document defines the trademark policy for OxideKit, including name usage rules, badge requirements, and fork guidelines.

---

## Overview

The OxideKit name, logo, and associated branding are trademarks that identify the official OxideKit project. This policy ensures:
- Users can trust software bearing the OxideKit name
- The community can build on OxideKit without confusion
- Forks and derivatives are clearly distinguished from official releases

## Name Usage Rules

### Permitted Uses (No Permission Required)

You MAY use the OxideKit name without permission when:

1. **Descriptive reference**: "This application is built with OxideKit"
2. **Comparative statements**: "Unlike OxideKit, our framework..."
3. **News and commentary**: Writing about OxideKit in articles, blogs, reviews
4. **Educational content**: Teaching about OxideKit in courses, tutorials
5. **Package metadata**: Declaring OxideKit as a dependency

### Permitted Uses (With Compliance)

You MAY use the OxideKit name with compliance requirements when:

1. **Plugin/extension names**: "MyPlugin for OxideKit" (not "OxideKit MyPlugin")
2. **Integration names**: "OxideKit integration for [Product]"
3. **Community resources**: "Awesome OxideKit", "OxideKit Users Group"

**Requirements**:
- Include a clear disclaimer: "Not affiliated with or endorsed by the OxideKit project"
- Do not use the OxideKit logo
- Do not imply official status

### Prohibited Uses

You MAY NOT:

1. **Use "OxideKit" as a product name**: "OxideKit Pro", "OxideKit Enterprise", "OxideKit Plus"
2. **Use in domain names**: oxidekit-pro.com, myoxidekit.io
3. **Imply official endorsement**: "Official OxideKit Partner" (without agreement)
4. **Use the logo without permission**: In products, marketing, or merchandise
5. **Create confusingly similar names**: "OxideKite", "Oxide-Kit", "OxKit"

## Badge Programs

### "Built with OxideKit" Badge

**Purpose**: Indicates an application uses OxideKit as its runtime.

**Requirements**:
- Application must use official OxideKit runtime
- Application must pass basic compatibility tests
- Self-certification (no approval required)

**Usage**:
```
Built with OxideKit
```

### "OxideKit Compatible" Badge

**Purpose**: Indicates a plugin/extension is compatible with OxideKit.

**Requirements**:
- Plugin must pass the OxideKit compatibility test suite
- Plugin must use official OxideKit APIs
- Plugin must declare required OxideKit version
- Self-certification with test results

**Usage**:
```
OxideKit Compatible - v1.x
```

### "Verified by OxideKit" Badge

**Purpose**: Indicates verified security and compatibility through the attestation service.

**Requirements**:
- Must use OxideKit Attestation Service
- Must pass security scanning
- Must have valid SBOM
- Must pass capability verification
- Issued automatically upon passing verification

**Usage**:
```
Verified by OxideKit
Attestation ID: [unique-id]
Valid until: [date]
```

**Verification**: Badge includes QR code linking to verification page.

## Rules for Forks

### Naming Requirements

Forks of OxideKit MUST:

1. **Use a different name**: Not "OxideKit [Something]" or "[Something] OxideKit"
2. **Remove OxideKit branding**: Logo, badges, official references
3. **Credit the origin**: "Originally derived from OxideKit"

### Acceptable Fork Names

- Use a completely different name (e.g., "RustFrame", "NativeCore")
- Use your company/project name (e.g., "AcmeRuntime")

### Unacceptable Fork Names

- "OxideKit Community Edition"
- "OxideKit Open Source"
- "Open OxideKit"
- "OxideKit Core" (unless official)
- Any name containing "OxideKit" or confusingly similar

### Fork Restrictions

Forks MUST NOT:

1. Claim to be "the real OxideKit"
2. Suggest official endorsement
3. Use OxideKit verification badges
4. Claim compatibility with official OxideKit ecosystem
5. Use OxideKit in marketing as primary identifier

## Enforcement

### Philosophy

OxideKit takes a **community-friendly approach** to trademark enforcement:

1. **Education first**: We contact and explain before taking action
2. **Good faith matters**: Unintentional violations get guidance, not threats
3. **Proportional response**: Enforcement matches the severity of violation
4. **Protect users**: Primary goal is preventing user confusion

### Enforcement Priorities

**High Priority** (immediate action):
- Malware or scams using OxideKit name
- Products marketed as official OxideKit offerings
- Use of "Verified by OxideKit" badge without verification

**Medium Priority** (contact and request change):
- Confusingly named forks
- Unauthorized logo use in products
- Domain name squatting

**Low Priority** (monitor and educate):
- Community projects with good intentions
- Minor naming guideline violations
- Educational/commentary uses

### Reporting Violations

Report trademark concerns to: trademark@oxidekit.com

Include:
- URL or location of violation
- Screenshot or evidence
- Brief description of concern
